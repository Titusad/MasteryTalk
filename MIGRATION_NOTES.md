# Migration from Figma Make to Standalone Repo

## Steps after cloning

### 1. Install dependencies
```bash
pnpm install
```

### 2. Environment variables
```bash
cp .env.example .env
# Fill in your Supabase credentials
```

### 3. Handle `utils/supabase/info.tsx`

This file was auto-generated by Figma Make with hardcoded credentials.
You have two options:

**Option A: Keep it as-is** (simplest)
The file already has your Figma Make project's Supabase credentials.
`src/services/supabase.ts` checks `.env` vars first, then falls back to this file.

**Option B: Replace with env-only** (cleaner for production)
Replace the contents of `utils/supabase/info.tsx` with:
```tsx
export const projectId = ""
export const publicAnonKey = ""
```
Then all credentials come exclusively from `.env`.

### 4. Google OAuth Setup
1. Google Cloud Console > Create OAuth 2.0 credentials
2. Authorized redirect URI: `https://{your-project-id}.supabase.co/auth/v1/callback`
3. Supabase Dashboard > Authentication > Providers > Google > paste Client ID + Secret

### 5. Deploy Edge Functions
The server code in `supabase/functions/server/` needs to be deployed as a Supabase Edge Function.
In Figma Make this is automatic. For standalone:
```bash
supabase functions deploy server
```

### 6. KV Store Table
The `kv_store` table is auto-created by Figma Make. For standalone Supabase,
run the SQL in your Supabase SQL editor:
```sql
CREATE TABLE IF NOT EXISTS kv_store_08b8658d (
  key TEXT PRIMARY KEY,
  value TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```
